# 研发路线

## 近期目标

项目近期目标是从文档基线进入可运行 ROS2 Humble 工作区，并逐步完成硬件、仿真、感知、导航、采样和端到端联调。

团队分工、接口责任和分支策略见 [team_workflow.md](team_workflow.md)。研发推进时按成员负责的可验收能力检查进度，而不是按代码量检查进度。

## 阶段 0: 初始化

目标:

- 建立 README、协作规则和核心设计文档。
- 建立 ROS2 Humble 单仓库工作区骨架、接口包、bringup 包、description 包和任务状态机包。
- 建立 7 人任务划分、跨模块接口约定和代码分支策略；当前分为移动自主组、视觉感知组和机械臂组。
- 明确赛题硬约束、平台基线和一级风险。
- 记录待归档资料来源。

验收:

- 文档可以支撑后续代码、依赖和硬件任务拆分。
- 每个成员有明确主责、交付物、接口消费者和验收标准。
- `colcon build --symlink-install` 在 Ubuntu 22.04 + ROS2 Humble 环境中可通过。
- 未验证事实均标注待验证、待归档或待称重。

## 阶段 1: 硬件 Bringup

目标:

- 在 Ubuntu 22.04 + ROS2 Humble + Python 3.10 环境中启动底盘、机械臂和传感器。
- 锁定 `scout_ros2` 与 `piper_ros` humble 的来源、分支和 commit。
- 建立基础 TF、URDF 和启动文件。

验收:

- 底盘可发布里程计并响应低速 `cmd_vel`。
- PiPER 可发布关节状态并执行安全预设动作。
- RGB-D、2D LiDAR、IMU 数据稳定发布。
- 完成基础 rosbag 和 bringup 记录。

责任人:

- 胡凯、李雨桐、吕思颐负责移动自主相关 bringup 输入，包括底盘接口联调、LiDAR、IMU、TF、基础 SLAM 和定位建图数据链路。
- 胡晟源、唐浩松负责视觉感知相关 bringup 输入，包括 RGB-D 图像采集、深度数据、相机内参和补光条件记录。
- 吕森炜、陈秉民负责 PiPER 上电、关节状态、末端执行器和安全预设动作。

## 阶段 2: 仿真

目标:

- 建立简化机器人模型、场地模型和科学目标模型。
- 在仿真中验证状态机、Nav2 参数和采样动作流程。
- 尽量复用真实 topic 和 launch 结构。

验收:

- 可在仿真中完成离开基地、搜索点巡航、接近目标、返回基地的流程。
- 任务状态机具备超时、失败和恢复路径。

## 阶段 3: 目标识别

目标:

- 构建科学目标检测与 3D 位姿估计流程。
- 评估补光灯对识别稳定性的影响。
- 输出可抓取性判断。

验收:

- 在典型距离和角度下稳定输出目标类别、置信度和位姿。
- 能处理漏检、误检、遮挡和深度异常。
- 识别结果可被导航和操控模块消费。

责任人:

- 胡晟源、唐浩松负责目标检测 / 分类 / 分割、补光策略、置信度输出，以及与 3D 位姿和抓取候选相关的感知接口。
- 视觉感知组需要与移动自主组确认目标接近点所需字段，并与机械臂组确认抓取候选所需字段。

## 阶段 4: 导航与越障

目标:

- 完成 slam_toolbox、robot_localization、Nav2 参数调试。
- 建立基地、搜索区、目标接近点和放置区的导航策略。
- 针对比赛场地障碍和光照条件调试局部避障。

验收:

- 机器人能稳定离开基地并到达搜索点。
- 能在 600 秒约束下规划返回基地路径。
- 导航失败时能执行有限恢复动作。

责任人:

- 胡凯、李雨桐、吕思颐负责 SLAM、探索策略、路径规划、局部避障、运动控制和返回基地策略。
- 移动自主组需要保障底盘、里程计、TF、雷达和定位输入稳定，并向系统集成交付导航状态和失败恢复记录。

## 阶段 5: 采样

目标:

- 建立 PiPER 预抓取、抓取、抬升、收回和放置动作序列。
- 验证末端执行器对科学目标的夹持稳定性。
- 将目标位姿转换为机械臂抓取位姿。

验收:

- 机械臂能在静态目标上重复完成抓取和放置。
- 底盘与机械臂动作互锁，避免移动中误操作。
- 抓取失败后有重试和放弃策略。

责任人:

- 吕森炜、陈秉民负责 PiPER 抓取、携带和放置动作。
- 机械臂组需要与视觉感知组确认目标 3D 位姿、抓取候选和可采样标记的接口格式。

## 阶段 6: 系统联调

目标:

- 打通离开基地、搜索、识别、取样、返回、放置全流程。
- 录制端到端 rosbag、视频和状态机日志。
- 固化比赛参数和启动流程。

验收:

- 至少完成多次端到端闭环演练。
- 每次演练记录耗时、失败点、恢复动作和改进项。
- 完成最终重量、尺寸、电源和安全检查。

## 阶段 7: 比赛材料

目标:

- 整理提交物、部署说明、现场检查单和备份方案。
- 归档官方资料、第三方依赖和关键参数版本。
- 准备两次出发机会的快速复位流程。

验收:

- 提交包内容完整。
- 现场人员可按检查单完成开机、校验、启动、复位和日志保存。
- 所有“待归档”“待称重”“待验证”项在赛前关闭或明确风险接受。
