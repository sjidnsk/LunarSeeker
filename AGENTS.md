# AGENTS.md

本文件定义本仓库内 Codex / agent 协作的固定规则。除非用户在当前任务中明确覆盖，否则所有 agent 必须遵守。

## 沟通语言

- 默认使用中文回答。
- 用户明确要求其他语言时，按用户指定语言回答。
- 文档和代码注释默认使用中文，除非项目上下文要求英文接口名、协议名或上游原文。

## 项目技术基线

- 系统固定为 Ubuntu 22.04。
- ROS 固定为 ROS2 Humble。
- Python 固定为 Python 3.10。
- 机器人平台基线为 SCOUT MINI + PiPER。
- 主要 ROS 组件基线包括 Nav2、slam_toolbox、robot_localization、ros2_control。
- 不要在未确认兼容性的情况下引入其他 ROS 发行版、Python 大版本或系统版本假设。

## 文件管理与危险操作

- 默认禁止批量删除文件或目录。
- 禁止使用以下命令或等价操作:
  - `del /s`
  - `rd /s`
  - `rmdir /s`
  - `Remove-Item -Recurse`
  - `rm -rf`
- 需要删除文件时，只能一次删除一个明确路径的文件，例如:

```powershell
Remove-Item "D:\path\to\file.txt"
```

- 如果确实需要批量删除文件或目录，必须停止并请求用户手动确认或手动删除。
- 递归移动、批量移动、大范围覆盖也按危险操作处理；执行前必须确认目标路径明确且范围受控。
- 允许读取文件、搜索文本、列目录、查看状态、运行测试和 dry-run 命令。

## 下载与临时资源位置

- 默认将下载资源、安装包、数据集、模型、缓存、导出文件等保存到 D 盘。
- 推荐路径:

```text
D:\CodexDownloads
```

- 不要默认把大文件、临时下载、数据集、模型、构建产物、Node cache、训练输出放在 C 盘。
- 小型测试产物、pytest cache、项目 ignored output 目录可以按工具默认行为生成，但不应提交进 Git。
- 如果 D 盘目录不存在，应优先创建明确的 D 盘目录，或在回复中说明准备使用的路径；不要静默改写到 C 盘。
- 如果由于工具或系统限制只能写入 C 盘，必须说明完整路径、用途、是否可以删除、建议删除时机。

## Git 与用户改动保护

- 默认认为工作区中已有改动可能来自用户或其他任务。
- 禁止执行:
  - `git reset --hard`
  - `git checkout -- <path>`
  - 大范围 revert
  - 清理未跟踪文件的批量命令
- 除非用户明确要求，不要回退、覆盖或删除已有改动。
- 提交时只纳入当前任务相关文件。
- 分支命名、用途和合入条件以 `docs/team_workflow.md` 的“开发分支”章节为准；agent 不应擅自创建、切换或重命名分支，除非用户明确要求。
- 发现无关 dirty 文件时应忽略。
- 如果无关改动影响当前任务，应先说明风险再继续。

## 文档与来源管理

- 赛题 PDF、平台官方资料、第三方仓库和不可读取链接都必须记录来源状态。
- 对未完成实测、未完成归档或未验证的内容，必须明确标注“待测量”“待归档”“待验证”。
- 重量、尺寸、续航、抓取成功率、识别准确率等指标不得凭经验写成已满足。
